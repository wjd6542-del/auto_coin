# 구현 문서 — 코인 자동매매 봇

> 이 봇이 **어떻게 만들어졌는지** (구조·모듈·기술스택)를 정의한다. 태스크 단위 구현 절차는 [plans](superpowers/plans/2026-07-17-coin-trading-bot-milestone1.md) 참고.

## 1. 기술 스택

| 분류 | 선택 | 비고 |
|------|------|------|
| 언어 | Python 3.13 (3.11+) | 웹 프레임워크 없음 |
| 데이터 처리 | pandas | 지표 계산·캔들 처리 |
| HTTP | requests | 빗썸 공개 API 호출 |
| DB/ORM | SQLite + SQLAlchemy 2.x | 추후 PostgreSQL 전환 가능 |
| 대시보드 | Streamlit | 수익률·거래내역 조회 |
| 테스트 | pytest | TDD |

- 외부 지표 라이브러리(TA-Lib 등) **미사용** — pandas로 직접 계산 (바이너리 의존성 회피).

## 2. 디렉토리 구조


## 3. 핵심 설계 원칙

**전략·리스크·데이터·실행을 독립 모듈로 분리한다.**
- 동일한 `strategy/`·`risk/` 코드를 백테스트·페이퍼·실거래 세 모드에서 그대로 재사용.
- 백테스트에서 검증된 로직이 실거래에서 다르게 동작하는 사고를 방지.
- 각 파일은 하나의 책임만 가지며 명확한 인터페이스로 통신.

## 4. 모듈별 인터페이스

### `config.py`
- `Settings` dataclass — 모든 전략 파라미터 (전략 문서 8절 참고).

### `bithumb/client.py`
- `BithumbClient(session=None)` — 세션 주입 가능(테스트용 mock).
  - `get_top_symbols(top_n, min_trade_value) -> list[str]`
  - `get_daily_candles(symbol) -> pd.DataFrame`
- 캔들 DataFrame: 컬럼 `[open, high, low, close, volume]`, `DatetimeIndex` 오름차순.
- ⚠️ 빗썸 캔들 배열 순서: `[ts, open, close, high, low, volume]` (open **다음이 close**).

### `strategy/indicators.py`
- `sma(series, period) -> pd.Series`
- `rsi(series, period=14) -> pd.Series`

### `strategy/signals.py`
- `Signal(action, reason)` — action: `"buy"`/`"sell"`/`"hold"`.
- `evaluate(candles, settings, in_position) -> Signal`

### `risk/manager.py`
- `Position(symbol, entry_price, qty, high_price)`
- `RiskManager(settings)` — `can_enter`, `position_size`, `update_high`, `hit_trailing_stop`.

### `db/models.py` + `db/store.py`
- 테이블: `trades`, `signals`, `balance_log`.
- `Store(db_path)` — `create_all`, `add_trade`, `add_signal`, `add_balance`, `trades_df`, `balance_df`.

### `engine/backtest.py`
- `BacktestResult(final_capital, return_pct, num_trades)`
- `Backtest(settings, store, fee_rate=0.0004).run(candles_by_symbol) -> BacktestResult`
- 날짜 오름차순 순회 → (1) 청산 검토 → (2) 진입 검토 → (3) 잔고 기록.

### `main.py`
- `run_backtest(client, store, settings) -> BacktestResult`
- `main()` — argparse로 모드 선택.

## 5. 데이터 흐름

```
[빗썸 API] → 일봉 수집 → [지표·신호] → [리스크: 진입 허용?]
    → [백테스트: 가상체결] → [DB 저장] → [Streamlit: 조회]
```

## 6. 실행 방법

```bash
# 백테스트 실행
python3 main.py --mode backtest

# 대시보드
streamlit run dashboard/app.py

# 테스트
pytest -q
```
