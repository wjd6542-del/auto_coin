# 구현 문서 — 코인 자동매매 봇

> 이 봇이 **어떻게 만들어졌는지** (구조·모듈·기술스택)를 정의한다. 태스크 단위 구현 절차는 `docs/superpowers/plans/`의 마일스톤별 계획 참고.

## 1. 기술 스택

| 분류 | 선택 | 비고 |
|------|------|------|
| 언어 | Python 3.13 (3.11+) | 웹 프레임워크 없음. `.venv` 가상환경(자동활성화 안 됨 → `.venv/bin/` 접두사) |
| 데이터 처리 | pandas | 지표 계산·캔들 처리 |
| HTTP | requests | 빗썸 공개 API 호출 |
| DB/ORM | **MySQL(운영) / SQLite(테스트)** + SQLAlchemy 2.x | PyMySQL 드라이버 |
| 대시보드 | Streamlit | 수익률·거래내역 조회 (모드 필터) |
| 테스트 | pytest | TDD |

- 외부 지표 라이브러리(TA-Lib 등) **미사용** — pandas로 직접 계산.

## 3. 핵심 설계 원칙

**전략·리스크·데이터·실행을 독립 모듈로 분리한다.**
- 동일한 `strategy/`·`risk/` 코드를 **백테스트·페이퍼·(추후)실거래** 모두에서 그대로 재사용.
- 백테스트에서 검증된 로직이 실거래에서 다르게 동작하는 사고를 방지.
- 각 파일은 하나의 책임만 가지며 명확한 인터페이스로 통신.

## 4. 모듈별 인터페이스

### `config.py`
- `Settings` — 전략 파라미터 (전략 문서 참고). `use_rsi_filter=False`(추세추종).
- `Secrets.from_env()` — 빗썸 API키 (`.env`에서 로드).
- `Database.from_env()` — `url()`로 MySQL/SQLite 접속 URL 생성.

### `bithumb/client.py`
- `BithumbClient(session=None)` — 세션 주입 가능(테스트 mock).
  - `get_top_symbols(top_n, min_trade_value) -> list[str]`
  - `get_daily_candles(symbol) -> pd.DataFrame`
- 캔들 DataFrame: 컬럼 `[open, high, low, close, volume]`, `DatetimeIndex` 오름차순.
- ⚠️ 빗썸 캔들 배열 순서: `[ts, open, close, high, low, volume]` (open **다음이 close**).

### `strategy/indicators.py`
- `sma(series, period) -> pd.Series`, `rsi(series, period=14) -> pd.Series`

### `strategy/signals.py`
- `Signal(action, reason)` — action: `"buy"`/`"sell"`/`"hold"`.
- `evaluate(candles, settings, in_position) -> Signal`
  - `use_rsi_filter=False`(추세추종): 골든크로스만으로 매수.
  - `use_rsi_filter=True`(보수적): 골든크로스 + RSI 회복.

### `risk/manager.py`
- `Position(symbol, entry_price, qty, high_price)` — 도메인 데이터클래스.
- `RiskManager(settings)` — `can_enter`, `position_size`, `update_high`, `hit_trailing_stop`.

### `db/models.py` + `db/store.py`
- 테이블: `trades`, `signals`, `balance_log`, `positions`(OpenPosition), `paper_account`, `app_settings`.
- `Store(db_path=None, url=None)` — 경로(SQLite) 또는 URL(MySQL).
  - 기록: `create_all`, `add_trade`, `add_signal`, `add_balance`, `trades_df`, `balance_df`.
  - 페이퍼 상태: `get_account`/`save_account`, `get_positions`/`add_position`/`remove_position`/`update_position_high`.
  - 설정: `get_settings() -> Settings`(없으면 config 기본값 초기화), `save_settings(settings)`.

### 설정 흐름 (DB 기반)
- 봇은 `config.settings`가 아니라 **`store.get_settings()`**로 파라미터를 읽는다.
- 대시보드 "⚙️ 전략 설정" 폼에서 수정 → `app_settings` 저장 → 다음 실행 반영.
- `max_volume_pct`: 종목당 투자금을 코인 거래대금(종가×거래량)의 이 비율 이하로 제한 → 슬리피지 통제. `position_size(capital, price, daily_value)`에서 적용.

### `engine/backtest.py`
- `BacktestResult(final_capital, return_pct, num_trades)`
- `Backtest(settings, store, fee_rate=0.0004).run(candles_by_symbol) -> BacktestResult`
- 날짜 오름차순 순회 → (1) 청산 → (2) 진입 → (3) 잔고 기록.

### `engine/tuner.py`
- `compute_mdd(balance) -> float` — 최대낙폭 %.
- `backtest_once(candles, settings, fee_rate) -> dict` — 인메모리 SQLite로 1회 백테스트.
- `run_grid(candles, base_settings, grid) -> list[dict]` — 조합별 결과, 수익률 내림차순.

### `engine/paper.py`
- `PaperTrader(settings, store, client, fee_rate=0.0004).run_once() -> dict`
- DB에서 현금·포지션 로드 → 청산 → 진입 → 잔고기록·저장. 체결가=최신 일봉 종가.

### `main.py`
- `fetch_candles(client, settings, min_len=None)` — 캔들 수집(재사용).
- `run_backtest`, `run_tune`, `run_paper`. `main()` — argparse `--mode backtest|tune|paper`.

## 5. 데이터 흐름

```
[빗썸 API] → 일봉 수집 → [지표·신호] → [리스크: 진입 허용?]
    → [백테스트/페이퍼: 체결] → [DB(MySQL): 저장] → [Streamlit: 조회]
```

## 6. 실행 방법

```bash
.venv/bin/python main.py --mode backtest   # 백테스트
.venv/bin/python main.py --mode tune       # 파라미터 튜닝(그리드)
.venv/bin/python main.py --mode paper      # 페이퍼 1 사이클
.venv/bin/streamlit run dashboard/app.py   # 대시보드 (localhost:8501)
.venv/bin/pytest -q                        # 테스트
./scripts/run_paper.sh                      # 페이퍼 cron 래퍼
```

**cron (매일 KST 00:30):** `30 0 * * * /Users/wjd/프로젝트/coin/scripts/run_paper.sh`
