# 코인 자동매매 봇 — 설계 문서

**작성일:** 2026-07-17
**상태:** 승인됨 (구현 계획 대기)

## 1. 목적

빗썸 거래소에서 **보수적 전략으로 안정적 수익**을 내는 자동매매 봇. 실거래 전에 백테스트로 전략을 검증하고, 페이퍼 트레이딩을 거쳐 소액 실거래로 넘어간다.

## 2. 확정 사항

| 항목 | 결정 |
|------|------|
| 언어 | Python (웹 프레임워크 없이, 순수 파이썬 + 라이브러리) |
| 거래소 | 빗썸 API |
| 전략 | 이동평균선 크로스 + RSI (두 조건 모두 만족해야 매수) |
| 타임프레임 | 일봉 |
| 매매 대상 | 빗썸 거래대금 상위 100종목 + 유동성 필터 |
| 손절 | 트레일링 스톱 |
| 자금 관리 | 분산 — 동시 최대 4종목, 종목당 총자산 20% |
| 진행 순서 | 백테스트 → 페이퍼 → 실거래 |
| 저장 | SQLite + SQLAlchemy (추후 PostgreSQL 전환 가능) |
| 화면 | Streamlit 대시보드 |
| 알림 | (미정 — 추후 텔레그램 등 옵션 검토) |

## 3. 매매 로직 (보수적 기본값)

### 매수 조건 (전부 만족해야 진입)
- **골든크로스:** 단기 이평선(20일)이 장기 이평선(60일)을 상향 돌파
- **RSI 회복:** RSI가 과매도(30) 아래로 갔다가 40 위로 복귀
- **유니버스 필터:** 거래대금 상위 100 + 최소 거래대금 기준 통과

### 매도 조건 (하나라도 해당 시 청산)
- **트레일링 스톱:** 진입 후 고점 대비 -5% 하락
- **데드크로스:** 단기 이평선이 장기 이평선을 하향 돌파

### 자금 규칙
- 동시 보유 최대 **4종목**, 4개 차 있으면 신규 매수 보류
- 종목당 총자산의 **20%**
- 같은 종목 중복 진입 금지

> **파라미터 기본값(20/60일, RSI 30/40, 트레일링 -5%, 4종목×20%)은 백테스트 튜닝의 시작값이다.** 설정 파일(`config.py`)에서 조정하며 최적값을 찾는다.

## 4. 시스템 구조

```
coin/
├── config.py          # 설정값·API키 (전략 파라미터 전부 여기)
├── bithumb/
│   ├── client.py      # 빗썸 API 래퍼 (시세조회, 잔고, 주문)
│   └── market_data.py # 상위100 종목 조회, 일봉 캔들 수집
├── strategy/
│   ├── indicators.py  # 이평선, RSI 계산
│   └── signals.py     # 매수/매도 신호 판단
├── risk/
│   └── manager.py     # 포지션 수·비중·트레일링스톱 관리
├── engine/
│   ├── backtest.py    # 백테스트 실행기
│   ├── paper.py       # 페이퍼 트레이딩 (실시세, 가짜주문)
│   └── live.py        # 실거래 실행기
├── db/
│   ├── models.py      # trades, signals, positions, balance_log
│   └── store.py       # 저장/조회 함수
├── dashboard/
│   └── app.py         # Streamlit 대시보드
└── main.py            # 봇 진입점 (모드 선택: backtest/paper/live)
```

### 핵심 설계 원칙
전략·리스크·주문실행을 **모듈로 분리**한다. 동일한 전략 코드(`strategy/`, `risk/`)를 백테스트·페이퍼·실거래 세 모드에서 그대로 재사용한다. 이렇게 하면 백테스트에서 검증된 로직이 실거래에서 다르게 동작하는 사고를 방지한다. 각 모듈은 하나의 책임만 가지며 명확한 인터페이스로 통신한다.

## 5. 데이터 흐름

```
[빗썸 API] → 일봉 수집 → [전략: 지표계산·신호] → [리스크 매니저: 진입 허용?]
    → [실행기: 주문] → [DB 저장] → [Streamlit: 수익률·거래내역 확인]
```

## 6. DB 스키마

| 테이블 | 저장 내용 |
|--------|-----------|
| `trades` | 매수/매도 체결 (시각, 코인, 가격, 수량, 수수료, 모드) |
| `signals` | 신호 발생 근거 (RSI값, 이평선 상태 — 진입/청산 사유) |
| `positions` | 현재 보유·평단·트레일링 최고가 |
| `balance_log` | 시점별 총자산 (수익률 그래프용) |

## 7. 개발 순서

1. **빗썸 API 연동 + 데이터 수집** — 상위100·일봉 캔들 조회
2. **지표·신호 로직** — 이평선/RSI 계산, 매수매도 판단
3. **백테스트 엔진** — 과거 데이터로 수익률 검증 (전략 튜닝 지점)
4. **DB + 저장 계층**
5. **Streamlit 대시보드**
6. **페이퍼 트레이딩** — 실시세로 무자본 검증
7. **실거래** — 검증 완료 후 소액부터

## 8. 리스크 / 주의사항

- **API 키 보안:** 실거래 키는 `config.py`에 하드코딩하지 않고 환경변수/별도 비공개 파일로 관리. `.gitignore` 필수.
- **실거래는 최후 단계:** 백테스트·페이퍼로 충분히 검증되기 전에는 실거래 모드 비활성.
- **거래소 제약:** 빗썸 API 요청 rate limit, 최소 주문금액, 수수료(약 0.04~0.25%)를 로직에 반영.
- **일봉 한계:** 하루 1회 판단이므로 급락 시 대응이 느릴 수 있음 — 트레일링 스톱 체크 주기를 별도 고려.
