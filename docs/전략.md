# 전략 문서 — 코인 자동매매 봇

> 이 봇이 **무엇을 근거로 사고 파는지**를 정의한다. 파라미터 튜닝·전략 변경은 이 문서를 먼저 갱신한다.

## 1. 기본 철학

- **보수적 운영, 안정적 수익.** 잦은 매매보다 확실한 신호에만 진입한다.
- **두 지표 동시 만족** 시에만 매수 → 잘못된 진입(속임수 신호) 최소화.
- **손실은 짧게, 이익은 길게** → 트레일링 스톱으로 상승분은 따라가되 하락은 잘라낸다.

## 2. 매매 대상 (유니버스)

| 항목 | 값 |
|------|-----|
| 거래소 | 빗썸 (KRW 마켓) |
| 종목 범위 | 24h 거래대금 **상위 100종목** |
| 유동성 필터 | 최소 거래대금 `min_trade_value_krw` (기본 10억 KRW) 이상 |
| 제외 | 저유동성 잡코인 (필터 미달분) |

## 3. 타임프레임

- **일봉(1일봉)** 기준. 하루 1회 판단.
- 노이즈·수수료 영향이 적고 보수적 성향에 부합.

## 4. 지표

| 지표 | 기본 파라미터 | 용도 |
|------|--------------|------|
| 단기 이동평균선 (SMA) | 20일 | 골든/데드크로스 판단 |
| 장기 이동평균선 (SMA) | 60일 | 골든/데드크로스 판단 |
| RSI | 14일 | 과매도 반등 확인 |

## 5. 매수 조건

전략은 `use_rsi_filter` 설정으로 두 모드를 전환한다.

**추세추종 모드 (기본, `use_rsi_filter=False`) — 현재 활성:**
1. **골든크로스** — 전일 단기 SMA ≤ 장기 SMA, 당일 단기 SMA > 장기 SMA
2. **유니버스 통과** — 상위 100 + 유동성 필터

추세 시작(골든크로스)에 진입해 데드크로스까지 추세 전체를 보유한다. 시장에 오래 머물러 상승장 수익을 극대화한다. (백테스트 12.5년 연 15.3%, MDD -26.7%)

**보수적 모드 (`use_rsi_filter=True`):**
1. 골든크로스 **AND**
2. **RSI 회복** — 최근 `rsi_period` 구간에서 RSI가 `rsi_oversold`(기본 30) 아래를 찍고 현재 RSI가 `rsi_recover`(기본 40) 초과
3. 유니버스 통과

진입 기회가 드물어 대부분 현금 보유. 안전하지만 수익 낮음 (백테스트 연 3.3%).

## 6. 매도 조건 (하나라도 해당 시 청산)

1. **트레일링 스톱** — 진입 후 고점 대비 `trailing_stop_pct`(기본 5%) 하락
2. **데드크로스** — 전일 단기 SMA ≥ 장기 SMA, 당일 단기 SMA < 장기 SMA

## 7. 자금 관리

| 규칙 | 기본값 |
|------|--------|
| 동시 보유 종목 수 | 최대 4종목 |
| 종목당 비중 | 총자산의 20% |
| 중복 진입 | 같은 종목 재진입 금지 |
| 슬롯 초과 시 | 매수 신호 많으면 심볼 알파벳 순 진입 (결정적) |

## 8. 파라미터 기본값 (튜닝 시작점)

파라미터는 **DB `app_settings` 테이블**에 저장되고 봇이 실행 시 읽는다. **대시보드 "⚙️ 전략 설정" 폼에서 수정**하면 다음 실행부터 반영된다. (DB가 비면 `config.py` 기본값으로 초기화) **아래는 백테스트로 최적화한 현재값**이며 고정값이 아니다.

| 파라미터 | 기본값 |
|----------|--------|
| `short_period` | 15 |
| `long_period` | 60 |
| `rsi_period` | 14 |
| `rsi_oversold` | 30 |
| `rsi_recover` | 40 |
| `use_rsi_filter` | False (추세추종) |
| `trailing_stop_pct` | 0.10 (10%) |
| `max_positions` | 4 |
| `position_pct` | 0.20 (20%) |
| `max_volume_pct` | 0.01 (1%) — 종목당 투자금을 코인 거래대금 대비 상한 |
| `top_n` | 100 |
| `min_trade_value_krw` | 1,000,000,000 |

## 9. 검증 절차

전략은 **백테스트 → 페이퍼 → 실거래** 순으로만 실전 투입한다.

1. **백테스트** — 과거 일봉으로 수익률·MDD·승률 확인. 파라미터 튜닝.
2. **페이퍼** — 실시세 가상매매로 로직/타이밍 검증.
3. **실거래** — 검증 완료 후 소액부터.

## 10. 수수료·제약 반영

- 빗썸 현물 수수료(약 0.04~0.25%)를 체결가에 반영 (백테스트 기본 `fee_rate=0.0004`).
- 최소 주문금액, API rate limit은 실행 엔진에서 처리.

## 11. 향후 전략 확장 후보 (미적용)

- 다중 타임프레임 필터 (일봉 추세 + 4시간봉 진입)
- 분할매수(DCA)로 평단 관리
- 변동성 기반 포지션 사이징
