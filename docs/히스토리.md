## 결정 기록 (Decision Log)

### 2026-07-17 — 프로젝트 착수 및 설계 확정

브레인스토밍을 통해 아래 사항을 확정했다.

| 결정 | 선택 | 근거 |
|------|------|------|
| 언어 | Python (장고 X) | 백테스트·지표 라이브러리 생태계 우위, 봇은 웹서버 아님 |
| 거래소 | 빗썸 | 요구사항 |
| 전략 | 이평선 크로스 + RSI | 사용자 선택, 두 조건 동시 만족 = 보수적 |
| 타임프레임 | 일봉 | 노이즈·수수료 적음, 보수적 성향 부합 |
| 매매 대상 | 상위 100종목 + 유동성 필터 | (전종목 스캔에서 100개로 축소) |
| 손절 | 트레일링 스톱 | 상승분 추종 + 하락 방어 |
| 자금 관리 | 분산 (동시 4종목, 종목당 20%) | 몰빵 방지 |
| 진행 순서 | 백테스트 → 페이퍼 → 실거래 | 돈 잃기 전 전략 검증 |
| 저장 | SQLite + SQLAlchemy | 초기 단순, 추후 Postgres |
| 화면 | Streamlit | 대시보드 최소 비용 |

**논의 중 변경된 사항:**
- 매매 대상: 처음 "전 종목 스캔" → **"상위 100종목"**으로 축소 (보수적 성향과 균형).

**권한 설정:**
- `.claude/settings.local.json` 생성 — 쉘/파일/라이브러리설치/타 프로젝트 참조 전부 허용, 질의 생략.
